## What went well

- **The architecture's substitutable seams paid off.** Faithful mode is mostly *composition + mock adapters* — the `PriceOracle`, default-deny `Authorization`, outbox/saga, and ERC-3643 ports built in Epics 3–8 absorbed the new behaviour with no core rewrite (Winston: "the seams did the work").
- **Honesty held end to end.** Every mock is labelled; `GET /mode` reports real-vs-mocked **derived from the composed deps**, not a hardcoded string; an always-visible banner states it. Nothing pretends to be production.
- **Hard invariants were respected under pressure.** 9.4 unblocked the §11.4 single-side close *without* breaking Story 8.6 (absent adapter ⇒ still fail-closed) and *without* breaking 8.5 solvency (the house conserves per-side backing). 9.2 made authz subject-aware via `AsyncLocalStorage` rather than changing the gate arity across DONE packages.
- **Independent live verification caught what tests missed** (see Insights).

## What was hard / what we learned

- **Two sub-agents died mid-flight** (9.2 API socket error, 9.5 connection closed). Both had done substantial, green work; recovery meant the orchestrator inspecting the working tree, re-running the full gate, and finishing the gaps — never trusting a self-reported `done` after an abnormal exit.
- **9.5 shipped a real bug that unit tests passed over.** The covenant-breach injection forced a BREACH ratio against group **NAV**, which is `NA` when NAV≈0 — exactly the seeded demo's state. Unit tests used a positive-NAV scenario and went green; the **live smoke** (`PUT /operator/covenant-breach` then `GET /group-view`) returned `['NA']`, not `BREACH`. Fixed by deriving the forced breach against a guaranteed-positive denominator (gross footing).
- **`MintAuthorizationGate` is a zero-arg thunk** — making the real gate subject-aware needed an `AsyncLocalStorage` context wrapper, a non-obvious integration that's now documented.
- **Scope discipline on a DONE package** (9.4 touching `@rose/positions`) required an *optional* port so paper/8.6 behaviour is byte-for-byte unchanged when the adapter is absent.

## Key insights

1. **Live smoke is not optional — it's the safety net.** The single highest-value practice this epic was the orchestrator re-running the gate *and* exercising the real `serve.js` after every story. It caught the covenant NA bug, the EIP-55 owner-casing bug (earlier program), and confirmed async timing — none of which the green unit suite revealed.
2. **Additive, port-backed change is low-risk and fast.** Framing faithful mode as a sibling `ENGINE_MODE` over existing ports (not a rewrite) kept P0 untouched and every step shippable.
3. **Mock ≠ dishonest, *if* it's labelled and derived.** The `/mode` endpoint + banner make "real vs simulated" unambiguous — the demo gains fidelity without overclaiming.
4. **Sub-agent fragility is a process risk to design around.** Connection/socket deaths happened twice in six stories; the recovery discipline (verify tree → re-gate → finish) is now a known, repeatable play.
